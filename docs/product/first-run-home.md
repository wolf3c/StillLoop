# 新用户首次打开首页文档

## 目标

新用户首次打开 StillLoop 时，应快速理解三件事：

1. StillLoop 用来把用户带回当前任务。
2. 评估默认在本机完成。
3. 使用前需要授予必要权限并准备模型。

首次打开不是完整教程，不承担解释所有设置的职责。

## 首次打开屏幕顺序

当前首次启动屏幕顺序为：

1. 欢迎页。
2. 权限获取引导。
3. 模型准备。
4. 任务输入主页。

如果用户已完成初始设置，再次打开会优先回到任务输入主页；如存在权限或模型问题，则可通过顶部问题入口回到对应页面。

## 欢迎页

欢迎页主标题：

“分心时，我会轻轻把你带回当前任务”

欢迎页说明：

“先写下这段时间最想完成的一件事。之后我只在你偏离时轻轻提醒，所有判断都在本机完成。”

主按钮为“开始设置”。

欢迎页同时展示三条隐私原则：

- 默认在本机处理，不上传屏幕、摄像头或任务内容。
- 只在判断需要时提醒，不持续打扰。
- 专注摘要和评估事件保存在本机，用户可随时停止使用。

## 登录启动设置

欢迎页和权限页都包含“登录时启动 StillLoop”选项。该选项只影响菜单栏入口：

- 不自动开始专注。
- 不自动采集屏幕或摄像头。
- 不自动启动模型。

首次打开时展示该选项，是为了让用户明确 StillLoop 可以作为常驻菜单栏工具，但不会在未开始任务前运行专注采集。

## 权限引导

点击“开始设置”后进入权限页。权限页解释 StillLoop 仅在本机读取必要屏幕和摄像头状态，用于判断是否需要提醒，不保存截图或摄像头画面。

权限页必须处理三种情况：

- 权限已允许：展示已允许。
- 权限未请求：引导用户继续触发授权。
- 权限被拒绝或未生效：引导用户去系统设置修复。

## 模型准备

权限流程完成后进入模型准备页。默认选中“应用自带模型”。用户可选择：

- 下载并使用自带模型。
- 暂不下载，自带模型不可用时使用基础规则。
- 手动配置本地 HTTP 或在线模型服务。

首次体验的理想路径是使用自带模型；开发环境可通过 `STILLLOOP_SKIP_MODEL_DOWNLOAD=1` 跳过下载。

## 进入主页

模型准备完成或用户选择继续后进入任务输入主页。此时页面只要求用户输入一件当前任务。开始按钮在任务为空时禁用。

首次进入主页后，如果自带模型已下载且用户选择自带模型，应用会在主页后台预热自带模型，为首次评估减少等待。

## 首次任务开始

首次点击“开始专注”时，系统仍会重新检查权限状态。这样可以覆盖用户从系统设置返回、权限刚刚变化或权限仍未生效的情况。

如果权限不满足，应用回到权限页；如果模型未准备，应用弹出下载提示或回到模型设置。只有必要条件满足后才进入专注运行页。

## 首次打开成功标准

- 用户明白 StillLoop 的核心用途。
- 用户知道屏幕和摄像头只用于本机判断。
- 用户能完成权限和模型路径选择。
- 用户能输入任务并进入专注页。
- 如果系统权限或模型不可用，用户能看到明确修复入口。
