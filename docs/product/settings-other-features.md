# 设置页面其他功能文档

## 页面目标

设置页集中提供非专注运行态的系统配置、模型入口、反馈、开源许可和隐私状态说明。

本文不重复模型设置、权限页和开源许可页的细节，只描述设置页内的其他功能。

## 设置页入口

顶部导航在以下页面显示“设置”：

- 任务输入页。
- 专注运行页。
- 复盘页。

欢迎页、权限页、模型准备页、设置页、隐私页和开源许可页不显示设置按钮，避免嵌套导航。

## 页面结构

设置页当前包含：

- 标题“设置”。
- 登录时启动 StillLoop。
- 模型设置入口。
- 反馈与建议入口。
- 开源许可与模型信息入口。
- 隐私状态区。

页面使用滚动布局，最大内容宽度约 600，主要卡片宽度约 520。

## 登录时启动

功能：允许用户控制 StillLoop 是否登录时启动。

产品语义：

- 登录后只保留菜单栏入口。
- 不会自动开始专注。
- 不会自动采集屏幕或摄像头。
- 不会自动启动模型。

系统集成通过 `LaunchAtLogin` 管理。测试环境必须避免从 XCTest host 注册真实 login item。

## 模型设置入口

入口文案：

- 标题：模型设置。
- 说明：配置本地 HTTP 或线上 OpenAI-compatible 模型服务。

点击后进入模型准备页。虽然说明中提到本地 HTTP 和线上服务，页面默认仍可选择应用自带模型。

## 反馈与建议

点击“反馈与建议”会打开 sheet。

反馈类型：

- 问题。
- 建议。
- 疑问。
- 其他。

字段：

- 反馈正文。
- 联系方式，可选。
- “仅用于回复本次反馈”同意开关。

提交规则：

- 反馈正文不能为空。
- 联系方式为空时，联系同意开关自动关闭。
- 只有用户提供联系方式且开启同意时，才提交反馈 contact。
- 提交时附带当前 screen 和 modelSource。

反馈通过 TraceMind feedback API 提交，不作为普通 analytics event 发送。

## 隐私状态区

隐私区展示当前隐私和运行状态：

- 本地优先：默认使用应用自带模型；手动模型服务只在用户选择后使用。
- 截图或摄像头画面只在内存中压缩为轻量视觉信号，不保存原图。
- 专注摘要和评估事件保存在本机 `Application Support/StillLoop`；不保存图片、照片或截图。
- 开发诊断日志路径。
- 模型下载状态。
- 自带模型 runtime 状态。
- 当前评估路径状态。

隐私状态区是说明和诊断入口，不提供直接操作。

## 菜单栏行为

StillLoop 常驻菜单栏。菜单栏左键打开主窗口；右键显示菜单：

- StillLoop 正在本地运行。
- 显示 StillLoop。
- 退出。

关闭主窗口不会退出应用，只隐藏窗口。重新点击菜单栏、Dock 重新打开或系统 reopen 时，会再次显示主窗口。

## 系统状态处理

应用监听屏幕睡眠、唤醒、会话离开和恢复：

- 系统变为 inactive 时暂停专注采集和模型运算。
- 系统恢复后刷新权限状态，并在适用时恢复专注采集。

## 遥测事件

设置相关不应记录用户输入原文、联系方式或隐私内容到普通事件。当前普通事件包括：

- route_change。
- focus_session_started。
- focus_session_paused。
- focus_session_resumed。
- focus_session_ended。
- focus_nudge_shown。
- model_issue_detected。

反馈正文只通过反馈提交路径发送，并带用户显式提交动作。

## 验收标准

- 登录启动文案必须明确“不自动采集、不自动启动模型”。
- 反馈联系方式必须是显式可选和显式同意。
- 设置页不应变成二级权限页或复杂调试面板。
- 隐私状态必须与当前模型选择、runtime 状态和下载状态一致。
