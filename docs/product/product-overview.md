# StillLoop 总产品文档

## 产品一句话

StillLoop 是一个本地优先的 macOS 专注伙伴：用户写下当前任务后，应用观察本机上下文，在判断跑偏时用轻量提醒把用户带回任务。

产品口号：跑偏？回来。

## 目标用户

- 需要在电脑上完成写作、编码、设计、阅读、研究等深度工作的个人用户。
- 容易在浏览器、聊天工具、视频或其他应用中偏离当前任务的用户。
- 关心隐私，希望专注判断尽量在本机完成的用户。

## 核心价值

1. 让用户在开始前明确当前任务。
2. 在工作过程中识别是否还在推进任务。
3. 当偏离明显或停滞时，用短提醒恢复注意力。
4. 在结束后提供复盘，帮助用户看到本次专注里的状态、提醒和常用应用。
5. 默认使用本机模型和本地存储，降低用户对隐私泄露的顾虑。

## 当前产品范围

StillLoop 当前是 SwiftPM 结构的 macOS App。主窗口包含欢迎页、权限页、模型准备页、任务输入页、专注运行页、复盘页、设置页、隐私页和开源许可页。

应用同时提供菜单栏入口。登录启动时只保留菜单栏入口，不自动开始专注、不采集屏幕或摄像头、不启动模型。

## 核心用户路径

1. 新用户打开应用，看到欢迎页。
2. 用户确认本地处理与隐私原则，进入权限获取引导。
3. 用户授予屏幕录制和摄像头权限。
4. 用户选择应用自带模型，或手动配置模型服务。
5. 用户输入当前任务并开始专注。
6. 应用循环采集本机上下文并评估状态。
7. 应用在必要时展示 nudge 提醒。
8. 用户暂停、继续或结束任务。
9. 应用展示复盘，包括时长、状态、提醒、常用应用和可继续任务入口。

## 核心状态

专注评估输出以下状态：

- focused：当前可见内容支持任务。
- uncertain：信号不明确或只有弱关联。
- distracted：当前内容明显偏离任务，或注意力反复分裂且没有清晰进展。
- stuck：任务上下文存在，但没有可见推进。
- resting：短暂休息或非任务暂停。
- away：用户离开电脑或不在摄像头画面中。

## 主要模块

- AppModel：屏幕状态、会话生命周期、权限、模型配置、采集循环、评估循环和复盘生成。
- StillLoopView：所有 SwiftUI 页面和设置入口。
- MacLocalContextProvider：采集前台应用、窗口、浏览器标签、截图和摄像头帧。
- LLMFocusEvaluator：LLM 专注评估编排。
- TaskRelevantTargetEvaluator：长期前台目标是否属于当前任务的判断。
- BundledModelRuntime：应用自带 llama.cpp runtime 的启动、复用、探测和停止。
- OpenAICompatibleLLMEngine：OpenAI-compatible HTTP 或 Unix socket 请求。
- SessionStore：本地保存专注摘要和评估事件。
- StillLoopTelemetry：安全的产品事件和用户反馈提交。

## 隐私原则

- 默认本地优先，不上传用户屏幕、摄像头或任务内容。
- 截图和摄像头画面只在内存中压缩为评估输入，原图不写入磁盘。
- 浏览器 URL 会去除 query 和 fragment 后用于诊断或目标判断。
- 专注摘要和评估事件保存在 `Application Support/StillLoop`。
- 手动模型服务只在用户选择后使用；在线模型服务的 API Key 由用户输入。

## 非目标

- 不做多人协作、组织看板或员工监控。
- 不做网站屏蔽或应用强制拦截。
- 不做云端同步。
- 不把 StillLoop 定位为通用 AI 聊天工具。

## 成功指标

- 用户能在 1 分钟内完成首次设置并开始专注。
- 开始专注后，状态变化能反映真实工作上下文。
- 提醒短、轻、可恢复，不制造打扰感。
- 模型不可用时能清晰回退到基础规则，并引导用户修复模型设置。
- 复盘能帮助用户理解本次专注，而不是只给泛泛鼓励。
