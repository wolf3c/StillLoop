# HTTP local 模型文档

## 功能目标

HTTP local 模型允许高级用户把 StillLoop 连接到本机运行的 OpenAI-compatible 模型服务，例如 LM Studio 或用户自行启动的 llama.cpp server。

该能力是手动模型配置的一种，不是默认开发或默认用户路径。默认路径仍是应用自带模型。

## 入口

用户进入“模型准备”或设置页“模型设置”，选择：

- 模型来源：手动配置大模型。
- 服务类型：本地模型 HTTP 服务。

## 表单字段

本地 HTTP 模型表单包含：

- 服务根地址，例如 `http://127.0.0.1:8080`。
- 固定展示 `/v1` 后缀。
- 模型名称。

页面说明支持：

- OpenAI-compatible `/v1/models`。
- OpenAI-compatible `/v1/chat/completions`。

本地 HTTP 模式不展示 API Key 字段。

## URL 规则

用户在本地 HTTP 输入框中输入服务根地址。应用内部会把空路径或 `/` 补齐为 `/v1`。

示例：

- 输入 `http://127.0.0.1:8080`，实际使用 `http://127.0.0.1:8080/v1`。
- 输入 `http://127.0.0.1:8080/v1`，页面显示根地址 `http://127.0.0.1:8080`。
- 输入自定义路径时，保留用户输入。

## 连接检查

用户修改配置后，应用会在停止输入约 650ms 后自动检查。用户也可以点击“检查连接”。

检查内容：

1. 服务地址是否是有效 URL。
2. 模型名称是否为空。
3. `/v1/models` 是否可达。
4. 目标模型是否存在。
5. 最小 chat completion 是否成功。
6. 尽可能判断服务是否声明视觉能力。

检查成功后显示“模型可用”，并提示“服务可达；目标模型存在；最小推理成功”。如果视觉能力未声明，应用提示运行时会直接提交压缩截图和摄像头图片验证真实表现。

## 启用条件

只有在手动模型配置完整且连接检查成功后，`useLocalLLM` 才会被视为有效。配置会保存在 UserDefaults 中：

- 模型来源。
- base URL。
- 模型名称。
- 是否启用手动模型。

## 专注评估中的行为

本地 HTTP 模型启用后，专注评估会使用三个 OpenAI-compatible engine：

- 用户在场状态评估。
- 屏幕任务匹配评估。
- 任务进展评估。

这些请求会提交任务文本、必要元数据、压缩截图和压缩摄像头帧。原始图片不写入磁盘。

## 失败与回退

如果手动模型在专注期间失败：

- 当前评估回退到基础规则。
- 状态显示“当前评估：基础规则（手动模型：失败原因）”。
- 记录 `model_issue_detected`，issue 类型以 `manualModelInference` 开头。
- 空闲时路由回模型设置页。

常见失败包括：

- 服务未启动。
- URL 不是 `/v1` 兼容路径。
- 模型名称不存在。
- 服务不支持 chat completions。
- 响应不是可解析 JSON。
- 请求超时。

## LM Studio 使用方式

1. 在 LM Studio 中启动本地 server。
2. 确认 OpenAI-compatible API 开启。
3. 在 StillLoop 输入服务根地址，例如 `http://127.0.0.1:8080`。
4. 输入 LM Studio 暴露的模型名称。
5. 点击“检查连接”。
6. 成功后继续到任务输入页。

## 产品边界

- StillLoop 不分发用户手动配置的本地模型。
- StillLoop 不保证第三方模型许可证、能力或性能。
- 本地 HTTP 模型由用户负责启动、关闭和维护。
- 本地 HTTP 模式可能使用 TCP localhost，因此不应用于 Mac App Store 默认自带模型路径。

## 验收标准

- 本地 HTTP 输入体验应隐藏 `/v1` 复杂度。
- 配置修改后应自动检查。
- 继续前必须能发现明显不可用配置。
- 失败时要回退基础规则，而不是中断专注任务。
